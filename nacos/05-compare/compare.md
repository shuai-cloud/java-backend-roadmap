# 注册中心对比：Nacos vs Eureka vs Consul vs ZooKeeper

> 选型是微服务架构的关键决策。本文从多个维度对比主流注册中心，帮你理解各自的优劣和适用场景。

---

## 一、总体定位

| 组件 | 定位 | 典型场景 |
|------|------|----------|
| **Nacos** | 注册中心 + 配置中心（二合一） | Spring Cloud Alibaba、Dubbo、K8s |
| **Eureka** | 纯注册中心（Netflix） | Spring Cloud Netflix（已维护状态） |
| **Consul** | 注册中心 + 配置中心 + KV Store | 国外微服务、多语言、服务网格 |
| **ZooKeeper** | 分布式协调服务 | Hadoop、Kafka、Dubbo 老版本 |

---

## 二、详细对比

| 维度 | Nacos | Eureka | Consul | ZooKeeper |
|------|-------|--------|--------|-----------|
| **CAP 模型** | AP（默认）/ CP 可切换 | AP（可用性优先） | CP（一致性优先） | CP（一致性优先） |
| **健康检查** | 心跳 + TCP/HTTP 探测 | 心跳（默认 30s） | 多种检查（脚本、HTTP、TCP） | 心跳（Session） |
| **配置中心** | 内置，功能完善 | 无 | 内置，但功能较弱 | 无（需额外实现） |
| **多数据中心** | 支持 | 不支持 | 原生支持 | 不支持 |
| **服务管理 UI** | 美观、功能丰富 | 简陋 | 较好 | 无（需第三方工具） |
| **集群部署** | 简单（需 MySQL） | 简单（自包含） | 简单（自带 Raft） | 复杂（需独立 ZK 集群） |
| **性能** | 高（长轮询 + gRPC） | 中等（REST） | 高（Gossip + Raft） | 中等（ZAB 协议） |
| **社区活跃度** | 非常活跃（阿里） | 维护状态（Netflix） | 活跃（HashiCorp） | 活跃（Apache） |
| **语言生态** | Java 优先，多语言 SDK | Java 为主 | 多语言 | 多语言 |

---

## 三、选型建议

### 1. 选 Nacos 的场景
- 国内微服务项目，Spring Cloud Alibaba 生态。
- 需要注册中心 + 配置中心一体化。
- 需要 AP 模式（大多数微服务场景）。
- 需要可视化服务管理界面。

### 2. 选 Eureka 的场景
- 老项目维护，Spring Cloud Netflix 遗留系统。
- 新项目不建议选（Netflix 已宣布维护状态，不再开发新功能）。

### 3. 选 Consul 的场景
- 国外项目，多语言微服务（Go、Node.js 等）。
- 需要服务网格集成（Consul Connect）。
- 需要多数据中心支持。

### 4. 选 ZooKeeper 的场景
- 大数据生态（Hadoop、Kafka 等）。
- Dubbo 老版本（2.7 以下）的注册中心。
- 需要 CP 强一致性的分布式协调场景（如分布式锁、Leader 选举）。

---

## 四、为什么微服务注册中心不推荐 ZooKeeper？

- **CP 模型**：发生 Leader 选举时，ZK 会短暂不可用（几十秒），导致整个注册中心不可用，微服务无法发现服务。
- **写性能瓶颈**：ZK 写操作需要过半确认，大规模服务心跳会导致性能下降。
- **运维复杂**：需要独立集群，监控、扩缩容都比 Nacos/Consul 麻烦。
- **功能单一**：只做注册中心，没有配置管理、健康检查细化等。

---

## 五、面试常见问题

### Q1：Nacos 和 Eureka 有什么区别？
> Eureka 是 Netflix 的纯注册中心，AP 模型，功能单一，已进入维护状态。Nacos 是阿里开源的注册中心 + 配置中心，AP/CP 可切换，功能更丰富，社区活跃。新项目建议用 Nacos。

### Q2：Nacos 和 Consul 怎么选？
> 国内选 Nacos，国外选 Consul。两者功能相似，但 Nacos 的配置中心更完善，中文文档更好；Consul 的多数据中心和 KV Store 更强。

### Q3：Nacos 的 AP 和 CP 模式怎么理解？
> 默认 AP（可用性优先），适合服务发现——即使读到旧数据也比不可用好。如果需要强一致性（如配置中心、分布式锁），可以切换为 CP 模式。Nacos 通过 Raft 协议实现 CP。

---

## 小结

- 新项目国内首选 Nacos，国外首选 Consul。
- Eureka 已过时，ZooKeeper 不适合做微服务注册中心。
- 选型要考虑 CAP 模型、功能完整性、运维成本和团队技术栈。
