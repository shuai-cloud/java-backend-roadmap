# Nacos 简介与核心概念

## 什么是 Nacos？

Nacos（Dynamic Naming and Configuration Service）是阿里巴巴开源的一个更易于构建云原生应用的动态服务发现、配置管理和服务管理平台。它相当于 **Eureka + Config Server + Bus** 的结合体，但功能更强、性能更好、运维更简单。

### 核心功能
- **服务注册与发现**：服务启动时向 Nacos 注册，消费者通过 Nacos 获取服务地址列表，支持健康检查、负载均衡。
- **动态配置管理**：配置集中存储，支持动态刷新，无需重启应用即可生效。
- **服务管理**：查看服务列表、实例详情、健康状态，支持权重路由、保护阈值。

### 为什么选 Nacos？
- **国产、社区活跃**：中文文档完善，阿里云商业化支持。
- **功能二合一**：注册中心 + 配置中心，省一个组件。
- **支持 AP 和 CP 模式**：默认 AP（可用性优先），适合微服务；可切换 CP，适合一致性要求高的场景。
- **生态融合**：Spring Cloud Alibaba、Dubbo、K8s 等无缝集成。

## 核心概念

| 概念 | 说明 |
|------|------|
| **服务** | 一个或一组提供相同功能的实例集合，如 `order-service`。 |
| **实例** | 服务的一个具体运行单元，如 IP:Port。 |
| **命名空间** | 用于隔离不同环境（开发/测试/生产）或不同业务。 |
| **分组** | 同一命名空间下的进一步隔离，默认 `DEFAULT_GROUP`。 |
| **保护阈值** | 当健康实例比例低于阈值时，Nacos 会将不健康实例也返回给消费者，防止雪崩。 |
| **配置** | 应用的配置数据，如数据库连接、开关等。 |
| **Data ID** | 配置的唯一标识，通常为 `${prefix}-${profile}.${file-extension}`。 |
| **长轮询** | Nacos 客户端通过长轮询监听配置变更，实现实时刷新。 |

---

## 应用场景

- **微服务注册发现**：替代 Eureka、Consul。
- **配置中心**：替代 Spring Cloud Config、Apollo。
- **动态 DNS**：服务路由、流量控制。
- **服务治理**：健康检查、权重路由、优雅上下线。

---

## 小结

- Nacos 是一站式的服务发现和配置管理平台。
- 核心概念：服务、实例、命名空间、分组、Data ID。
- 国内微服务生态的首选，建议熟练掌握。
