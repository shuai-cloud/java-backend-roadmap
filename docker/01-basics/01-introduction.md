# Docker 简介与架构

## 什么是 Docker？

Docker 是一个开源的容器化平台，用于自动化应用程序的部署、扩展和管理。它允许你将应用程序及其依赖打包到一个轻量级、可移植的容器中，然后发布到任何 Linux 或 Windows 机器上。

### 与虚拟机的区别

| 特性 | Docker 容器 | 虚拟机 |
|------|------------|--------|
| 启动速度 | 秒级 | 分钟级 |
| 内核 | 共享宿主机内核 | 独立内核 |
| 资源占用 | 低（MB 级） | 高（GB 级） |
| 隔离性 | 进程级隔离 | 完全隔离 |
| 镜像大小 | MB 级 | GB 级 |
| 性能 | 接近原生 | 有一定损耗 |

## Docker 架构

Docker 使用 **C/S（客户端-服务器）架构**：

┌──────────────┐      REST API       ┌──────────────────┐

│  Docker CLI  │ ──────────────────→  │  Docker Daemon   │

│  (客户端)     │ ←──────────────────  │  (服务端)         │

└──────────────┘                      └────────┬─────────┘

│

┌─────────▼─────────┐

│   Registry (仓库)   │

│  Docker Hub / 私有  │

└───────────────────┘

- **Docker Daemon（dockerd）**：后台守护进程，管理镜像、容器、网络、数据卷。
- **Docker Client（docker）**：命令行工具，用户通过它与 Daemon 交互。
- **Docker Registry**：镜像仓库，默认 Docker Hub，也可搭建私有仓库。

## 核心概念

| 概念 | 说明 |
|------|------|
| **镜像（Image）** | 只读模板，包含运行应用所需的文件系统和配置。 |
| **容器（Container）** | 镜像的运行实例，可读可写，独立运行。 |
| **仓库（Repository）** | 存放镜像的地方，类似 Git 仓库。 |
| **Dockerfile** | 构建镜像的脚本文件。 |
| **数据卷（Volume）** | 持久化容器数据的机制。 |
| **网络（Network）** | 容器间通信的虚拟网络。 |

---

## 小结

- Docker 是轻量级容器化平台，比虚拟机更高效。
- 架构：Client → Daemon → Registry。
- 核心概念：镜像、容器、仓库、Dockerfile、数据卷、网络。
